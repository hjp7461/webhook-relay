# 00. Decisions Needed — 결정 필요 항목 (Open Questions Register)

> 본 문서는 PRD `07-out-of-scope-and-future.md` §2의 **모든 오픈 퀘스천 21건**을 그대로 가져와,
> PLAN 단계의 작업 의존성과 함께 한 곳에 모은 단일 출처다.
>
> **모든 항목은 "결정 보류" 상태다.** PLAN 본 묶음(`01-milestones.md` 이하)이 인용하는 권장 기본값은
> PRD에 기록된 잠정 가정(provisional)을 옮겨 적은 것이며, 사람의 명시적 승인 없이는 결정으로
> 단정하지 않는다. PLAN 구현 착수 전에 본 문서를 닫는 것이 선행 조건이다.
>
> **AI 협업 5원칙(CLAUDE.md §7) 적용:** 원칙 4(불확실하면 멈추고 묻는다) — 본 문서는 그 원칙의
> 산출물이다. 임의 결정 금지.

## 0. 사용법

- 각 항목은 PRD의 `Q-XXX-N` ID를 그대로 보존한다(추적성).
- "막히는 PLAN 항목"은 본 결정이 잠기지 않으면 착수할 수 없는 마일스톤/단계를 가리킨다.
- "권장 기본값"은 PRD의 Provisional을 그대로 따른다. **결정이 아니다.**
- 사람의 결정이 끝나면 본 문서에서 `Status: Resolved (YYYY-MM-DD — 결정 결과 요약)`로 표기하고
  PRD `07`의 동일 Q에 동일 결과를 반영한다.

## 2. 결정 항목 전체 (PRD `07` §2 순서 보존)

### 2.1 API / 인증 / 인터페이스 (4건)

#### Q-API-1 — 작업 등록 API 인증 방식
- **선택지:**
  - (a) 인증 없음 (데모용 명시)
  - (b) 공유 시크릿 (`Authorization: Bearer <SECRET>`)
  - (c) HMAC 요청 서명 (수신용 HMAC과 별도 키)
- **트레이드오프:**
  - (a) — 구현 단순. 외부 노출 시 위험. README 경고 필수.
  - (b) — 운영 안전. 환경변수 하나 추가. 키 회수 로직은 없음.
  - (c) — 가장 안전. 구현 비용 가장 큼. 본 PRD의 범위 N1.5(인증/인가 비목표)와 충돌 위험.
- **권장 기본값:** **(a) 인증 없음** — README/대시보드에 "데모/로컬 전용" 명시.
- **막히는 PLAN 항목:** M2 `POST /webhooks` 라우트 핸들러 설계, M2 셧다운 응답 정책.
- **Status:** Resolved (2026-05-26 — **(b) 공유 시크릿 Bearer** 선택). 환경변수 `API_BEARER_TOKEN`(필수, 최소 32 bytes, Zod fail-fast)로 검증. `Authorization` 헤더 누락/불일치 시 `401`. 키 회수/롤테이션은 본 PRD 범위 밖. **PRD 갱신 필요:** `05-api-and-contracts.md` §3 비목표·§6 경계 진술·요청 스키마 주석을 본 결정에 맞게 수정해야 함(다음 일괄 갱신 시 반영).

#### Q-API-2 — 멱등성 재요청 응답 코드
- **선택지:**
  - (a) `202 Accepted` + 동일 `jobId`
  - (b) `409 Conflict` + 기존 `jobId`
- **트레이드오프:**
  - (a) — 클라이언트 단순. "idempotent re-submit"의 일반적 시맨틱.
  - (b) — 명시적이지만 클라이언트가 `409`를 에러로 오인할 수 있음.
- **권장 기본값:** **(a) `202` + 동일 jobId**.
- **막히는 PLAN 항목:** M3 멱등성 통합 테스트(`IT-S2`)의 응답 코드 단언, M3 라우트 핸들러 반환 분기.
- **Status:** Resolved (2026-05-26 — **(a) `202 Accepted` + 동일 `jobId`** 채택, 권장 기본값).

#### Q-API-3 — 요청 본문 `headers` 화이트리스트 정책
- **선택지:**
  - (a) 모두 허용 (단, `Authorization`/`Cookie`/`Host` 등 블랙리스트만 차단)
  - (b) 허용 목록 명시
- **트레이드오프:**
  - (a) — 유연. 시크릿 누출 위험(`06` 재검토 필요).
  - (b) — 안전. 데모 가치 일부 손실.
- **권장 기본값:** **(a) 블랙리스트만 적용**.
- **막히는 PLAN 항목:** M2 요청 Zod 스키마, M2 워커의 outgoing 헤더 빌더, M2의 로그 마스킹 정책 일부.
- **Status:** Resolved (2026-05-26 — **(a) 블랙리스트만 적용** 채택, 권장 기본값). 차단: `Authorization`, `Cookie`, `Host`, `Content-Length`, `Transfer-Encoding`(대소문자 무시).

#### Q-API-4 — API 응답에 작업 추적 필드 포함 여부
- **선택지:**
  - (a) 등록 응답은 `jobId`만. 추적은 별도 GET API(본 PRD 범위 밖)
  - (b) 등록 응답에 상태 일부 포함
- **트레이드오프:**
  - (a) — 단순. PRD `05` §4.3과 일관.
  - (b) — UX 좋음. PRD 비범위 확장.
- **권장 기본값:** **(a) `jobId`만 반환**.
- **막히는 PLAN 항목:** M2 응답 스키마.
- **Status:** Resolved (2026-05-26 — **(a) `jobId`만 반환** 채택, 권장 기본값).

### 2.2 에러 분류 / 재시도 정책 (3건)

#### Q-RETRY-1 — 3xx 응답의 분류
- **선택지:**
  - (a) `NonRetriableError` (자동 리다이렉트 미수행)
  - (b) 자동 1회 리다이렉트
- **트레이드오프:**
  - (a) — 보수적. SSRF/체인 우려 회피.
  - (b) — 호환성 좋음. 보안 위험 증가.
- **권장 기본값:** **(a) `NonRetriableError`** — 즉시 DLQ.
- **막히는 PLAN 항목:** M4 에러 분류 함수의 분기 케이스(`UT-2`), `IT-S5`의 입력 케이스.
- **Status:** Resolved (2026-05-26 — **(a) `NonRetriableError` 즉시 DLQ** 채택, 권장 기본값). 자동 리다이렉트 미수행(SSRF/체인 우려).

#### Q-RETRY-2 — 408/425/429의 분류
- **선택지:**
  - (a) 모두 `RetriableError`
  - (b) 408/425만 retriable, 429는 `Retry-After` 헤더 존중하는 특별 정책
- **트레이드오프:**
  - (a) — 단순. 결정론적 백오프와 일치.
  - (b) — 수신자 친화. 구현 비용 추가.
- **권장 기본값:** **(a) 모두 retriable** — `Retry-After` 존중은 향후 개선.
- **막히는 PLAN 항목:** M4 에러 분류 함수(`UT-2`).
- **Status:** Resolved (2026-05-26 — **(a) 408/425/429 모두 retriable** 채택, 권장 기본값). `Retry-After` 존중은 후속 PRD.

#### Q-RETRY-3 — 백오프 jitter 적용 여부
- **선택지:**
  - (a) BullMQ 기본 지수 백오프만
  - (b) jitter(±랜덤) 추가
- **트레이드오프:**
  - (a) — 결정론적 테스트 작성 용이. fake timer 단언이 정확히 일치.
  - (b) — 운영상 thundering herd 방지에 유리. 테스트 결정성 손상.
- **권장 기본값:** **(a) jitter 없음** — 본 PRD의 결정성 우선 원칙 일치.
- **막히는 PLAN 항목:** M4 BullMQ Worker 옵션 설정, `IT-S3` 백오프 단언식.
- **Status:** Resolved (2026-05-26 — **(a) jitter 없음** 채택, 권장 기본값). 결정성 우선(IT-S3 fake timer 단언 정합성).

### 2.4 보안 / SSRF / 시크릿 (6건)

#### Q-SEC-1 — SSRF 방어 (private CIDR/localhost 차단)
- **선택지:**
  - (a) 차단하지 않음 (데모가 `localhost`를 쓰므로 필수 예외)

  - (c) 차단 ON 기본, 데모는 별도 우회 경로
- **트레이드오프:**
  - (a) — 가장 단순. 운영 보안 위험.
  - (b) — 데모/운영 모두 만족. 환경변수 하나 추가.
  - (c) — 보안 우수. 데모 가치 일부 손상.
- **권장 기본값:** **(b) 환경변수 토글**.
- **막히는 PLAN 항목:** M2 워커 송신 사전 검증, `demo/config.ts` 환경변수 스키마.
- **Status:** Resolved (2026-05-26 — **(b) 환경변수 토글** 선택, 권장 기본값 채택). `ALLOW_PRIVATE_TARGETS` env(기본 `true`로 데모 동작 유지, 운영에서는 `false` 권장).

#### Q-SEC-2 — HMAC 서명에 timestamp/nonce 추가 (replay 방어)

  - (a) — 단순. 결정성 보존(재시도 시 동일 서명).
  - (b) — replay 방어. 결정성 손상(timestamp가 시도마다 변함) → `IT-S3` 검증 복잡화.
- **권장 기본값:** **(a) 미적용**.
- **막히는 PLAN 항목:** M4 HMAC 서명 모듈(`demo/domain/hmac.ts`)의 입력 시그니처.
- **Status:** Resolved (2026-05-26 — **(a) 미적용** 채택, 권장 기본값). 재시도 결정성 보존(`IT-S3`의 fake timer 단언).
